- 白屏时间
- FCP（First Contentful Paint）
  - 页面**第一次“有内容出现”**的时间
  - FCP 的关键是：别让浏览器“等资源”，尽快画出第一帧内容。
    - 👍 < 1.8s：很好 1.8 ~ 3s：还能接受 > 3s：偏慢
- LCP
  - 页面**最大内容元素（图片、文字等）首次出现**的时间
  - 最大的那块内容加载太慢，或被 JS / 布局阻塞
  - 大图/主标题/主内容区域
    - 👍 < 2.5s：很好 2.5 ~ 4s：还能接受 > 4s：偏慢
- TTI（Time to Interactive）
  - 页面**首次可交互**的时间
  - 👍 < 5s：很好 5 ~ 8s：还能接受 > 8s：偏慢
- FID（First Input Delay）
  - 用户第一次操作（点、输入） 浏览器多久才开始处理
- INP
  - 用户操作后，多久能看到页面反馈
  - 比如loading
  - 比如输入之后有文字
- CLS（布局抖动）
  - 页面加载过程中，内容突然跳来跳去
- FPS / 卡顿（动画 & 滚动）
  - 页面的流畅度，页面滑动、动画是否顺滑

# 具体优化手段

## CLS（布局抖动）

- 图片 / 视频：一定要预留尺寸（最重要）
- 骨架屏/占位符
- 字体。@font-face {font-display: swap}
- 动画 & 过渡只用 transform / opacity transform 不会触发布局重排

## fps

- 主线程太忙了，来不及画下一帧
- requestAnimationFrame 控制动画
- window.addEventListener('scroll', handler, { passive: true })
- 防抖节流
- 动画是否只用了 transform / opacity
- 虚拟列表
