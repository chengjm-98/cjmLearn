## 1. TypeScript 和 JavaScript 的区别

问题: TypeScript 和 JavaScript 有什么区别？

回答: TypeScript 是 JavaScript 的超集，提供了静态类型检查、类型注解、接口、泛型等功能。TypeScript 编译成 JavaScript 后执行，而 JavaScript 本身是一种动态类型语言。

## 2. TypeScript 的类型推导机制

问题: TypeScript 的类型推导是如何工作的？

回答: TypeScript 会根据变量的赋值来推导其类型。例如，let x = 10; TypeScript 会推导 x 的类型为 number。如果没有显式的类型注解，TypeScript 会根据上下文来推导类型。

## 3. 接口与类型别名的区别

问题: interface 和 type 有什么区别？什么时候使用它们？

回答:

interface 用于定义对象的结构，可以通过扩展和实现来继承。适用于描述对象类型和类。

type 用于定义类型别名，可以是任何类型，包括原始类型、联合类型、元组等。它不支持扩展和实现。

一般情况下，可以使用 interface 来描述对象类型，而 type 用于定义更复杂的类型。

## 4. 联合类型与交叉类型

问题: 什么是联合类型（|）和交叉类型（&）？举个例子。

回答:

联合类型 (|) 表示一个值可以是多种类型之一。例如：let a: string | number; 表示 a 可以是字符串或数字。

交叉类型 (&) 表示将多个类型合并为一个类型，要求值同时具有这些类型的所有特征。例如：type A = { a: number }; type B = { b: string }; type C = A & B; C 需要同时具有 a 和 b 属性。

## 5. 泛型

问题: TypeScript 中的泛型是什么？如何使用泛型？

回答: 泛型使得类型在使用时才会被指定，从而实现了类型的复用和灵活性。它允许在定义时不指定具体的类型，而在使用时才指定。例如：

```jsx
function identity<T>(arg: T): T {
  return arg;
}
const result = identity(10); // T 被推导为 number
```

## 6. 类型断言

- as 关键字与类型断言

## 问题: as 关键字与类型断言有何区别？

回答: as 关键字用于将一个值的类型显式地转换为另一个类型，称为类型断言。在 TypeScript 中，类型断言是告诉编译器某个值的类型，而不是进行类型转换。例如：
let x: unknown = "hello";
let y = x as string; // 将 x 断言为 string 类型
