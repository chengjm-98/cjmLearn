我理解的前端工程化，本质是用工程化手段来提升前端项目在开发效率、代码质量、交付稳定性和可维护性上的整体水平，它覆盖了从开发、构建、测试到部署发布的完整流程。

# 项目规范体系

- 在项目规范层面，我主要关注代码规范、提交规范和目录规范。
- 通常会用 ESLint + Prettier 做代码规范
- 前端规范，目录结构
- 分支规范/提交规范

# 构建与工具链

- 在构建层面，我主要使用 Vite 或 Webpack 作为构建工具，配合 Babel 做语法转译，PostCSS 处理样式兼容性。
- 我比较关注构建速度、按需加载、代码分割和缓存策略。
- 比如拆包、开启 gzip、优化 chunk 体积等。

# 多团队协作

- 微前端，模块联邦（webpack）

# 发布流程

- 码云/jenkins
- 测试
- 灰度发布
  我没有从 0 搭建过 Jenkins 服务，但我了解 Jenkins 的基本工作模式，包括拉代码、执行构建脚本、产物发布以及环境变量配置。如果需要我可以快速上手维护 Pipeline。

# 总结

我认为工程化不是单点工具，而是从规范、构建到发布的一整套协作体系。我虽然没有从 0 搭建过 CI/CD，但对完整交付流程是非常熟悉的。
