# 什么是 Canvas ？

Canvas 是 HTML5 引入的一种新的网页元素，它允许通过 JavaScript 绘制图形、图像和动画。它本身是一个区域，你可以在其中绘制任何图形，如线条、矩形、圆形、文本，甚至图像和视频帧。Canvas 提供了一种通过脚本进行图形绘制的方式，而不需要依赖图形库或者外部工具

# Canvas 与普通的 div 有什么区别？

- <canvas> 和 <div> 都是 HTML 元素，但它们的用途和特点有所不同。

## 渲染方式：

Canvas：是一个静态区域，你可以使用 JavaScript 动态绘制图形。所有的绘制都是通过脚本来完成的，不能直接通过 CSS 操作图形。

div：是一个容器元素，主要用于布局和显示内容，可以通过 CSS 和 JavaScript 进行样式和行为控制，支持文本、图片、表单等内容，但不直接支持图形绘制。

## 用途：

Canvas：用于绘制图形、图像、动画和游戏场景，适用于动态生成内容。

div：主要用于网页结构和布局，展示静态内容，如文字、图片、按钮等。

## 渲染方式：

Canvas：绘制的内容是位图（像素级）。一旦绘制完成，图形就变成了固定的像素数据，无法通过 DOM 操作直接访问和修改。需要通过脚本来重新绘制。

div：内容是基于 DOM 元素的，可以通过 CSS 样式直接控制，支持交互和响应式布局，能轻松地改变内容和样式。

## 交互性：

Canvas：Canvas 内的内容本身不支持交互，交互（如点击事件）需要通过 JavaScript 来处理。例如，点击 Canvas 上的某个图形，需要自己写代码来检测点击位置。

div：div 是标准的 DOM 元素，直接支持事件处理和交互（如 click、hover 等）。

## 性能：

Canvas：性能相对较高，尤其是在处理大量图形和动画时，因为它是位图渲染。对于复杂的图形和游戏，Canvas 是一个更合适的选择。

div：渲染性能较低，特别是当包含大量元素时。如果是动态生成很多元素（比如列表或图表），性能可能较差。使用 CSS 动画和 JavaScript 控制 DOM 会造成性能问题。

## 使用方式：

Canvas：需要通过 JavaScript 操作其上下文进行图形绘制，如 getContext('2d') 来获取 2D 绘图上下文。

div：是一个普通的 HTML 元素，可以直接在 HTML 中定义并通过 CSS 进行样式设置。
