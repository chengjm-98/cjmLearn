# 二分法查找

## 前提

- 前提：数组必须是有序的
- 二分查找的前提是数组必须是有序的，否则无法使用二分查找。
- 二分查找的时间复杂度是 O(log n)，其中 n 是数组的长度。
- 二分查找的空间复杂度是 O(1)，因为它只需要一个额外的变量来存储中间索引。

## 思路

- 二分查找的思路是每次将数组分成两半，然后判断目标元素是否在左边或者右边。
- 如果目标元素在左边，那么就继续在左边查找；如果目标元素在右边，那么就继续在右边查找。
- 直到找到目标元素或者数组为空。

## 代码

```js
function binarySearch(arr, target) {
  let low = 0;
  let high = arr.length - 1;

  while (low <= high) {
    const mid = Math.floor((low + high) / 2);

    // 找到目标元素
    if (arr[mid] === target) {
      return mid; // 返回目标元素的索引
    }

    // 目标元素小于中间值，搜索左半部分
    if (arr[mid] > target) {
      high = mid - 1;
    }
    // 目标元素大于中间值，搜索右半部分
    else {
      low = mid + 1;
    }
  }

  return -1; // 如果没有找到，返回 -1
}
```

# 哈希查找

## 使用 map

- 哈希查找的时间复杂度是 O(1)，因为它只需要一次查找。
- 哈希查找的空间复杂度是 O(n)，因为它需要一个额外的数组来存储哈希表。

```js
function hashSearch(arr, target) {
  const hashMap = new Map();

  // 构建哈希表
  for (let i = 0; i < arr.length; i++) {
    hashMap.set(arr[i], i);
  }
  // 查找目标元素
  if (hashMap.has(target)) {
    return hashMap.get(target); // 返回目标元素的索引
  }
  return -1; // 如果没有找到，返回 -1
}
```
