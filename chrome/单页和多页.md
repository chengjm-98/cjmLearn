# 单页和多页

## 单页 - 单页面应用（SPA）

### 定义：单页面应用是一种在一个网页上进行所有操作的应用。页面在加载时只加载一次，之后所有的交互和内容更新都不会触发完整的页面刷新。

### 特点：

- 初次加载时，页面会加载一个基本框架（HTML、CSS、JavaScript），后续的交互会通过 AJAX 或 Fetch 向后台请求数据并更新页面内容。

- 用户的操作不会导致页面重新加载，只有局部的视图更新，提升了用户体验。

- 页面切换时不会重新加载页面，因此速度较快，适合动态内容和频繁交互的应用。

## 多页 - 多页面应用（MPA）

### 定义

多页面应用是传统的网页应用架构，每次用户请求一个新页面时，都会重新加载整个页面。每个页面通常都是一个独立的 HTML 文件。

### 特点：

每次页面切换都会向服务器发起一个新的请求，服务器返回一个完整的新页面。

每个页面通常有自己的 URL，可以独立加载，并且每个页面都能完整地渲染。

# 比较

| 特性             | 单页面应用（SPA）                                      | 多页面应用（MPA）                              |
| ---------------- | ------------------------------------------------------ | ---------------------------------------------- |
| **页面刷新**     | 无页面刷新，只更新部分内容                             | 每次页面切换都会重新加载整个页面               |
| **用户体验**     | 流畅、快速，页面间切换无白屏                           | 页面间切换可能会出现白屏，体验较差             |
| **开发方式**     | 前后端分离，复杂的路由和状态管理                       | 后端通常负责渲染页面，前端处理交互             |
| **SEO**          | 不利，需配合 SSR 或 SSG 解决                           | SEO 友好，每个页面有独立的 URL，易于爬取和索引 |
| **首屏加载时间** | 可能较长，因为需要加载大量资源（JS、CSS）              | 加载速度较快，页面是完整的 HTML 和资源         |
| **适用场景**     | 需要频繁交互和动态内容更新的应用，如社交平台、SPA 应用 | 传统的多页面网站，如企业官网、电商、新闻网站等 |
