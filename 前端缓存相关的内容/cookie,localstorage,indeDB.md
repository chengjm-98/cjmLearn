# cookie

## 存储位置

浏览器中的一个文件，存储在用户的计算机上，通常在用户首次访问网站时创建，并且在用户关闭浏览器时过期。

## 特点

- 存储容量小：每个 cookie 的大小通常限制在 4KB 左右。
- 过期时间：可以设置 cookie 的过期时间，也可以让 cookie 在浏览器关闭时自动过期。

## 用途

- 会话管理：用于存储用户的会话信息，如登录状态 token、购物车信息等。
- 个性化设置：用于存储用户的个性化设置，如主题、语言等。
- 跟踪分析：用于存储用户的行为数据，如访问次数、访问路径等。
- 作用域：同源

## localhost:3000 和 localhost:5000 的 Cookie【默认不共享】。

- 必须要同源才会共享

# localStorage

## 特点

- 存储容量： 每个域名下大约可以存储 5MB（具体大小因浏览器而异）。
- 生命周期： 数据没有过期时间**永久存在**，除非显式删除，数据会一直存在。
- 作用域： 只能在同一个域名和协议下访问（不同子域名间不可共享）。

## 用途

用来存一些长期有效的数据

- 用户偏好设置： 存储用户的个性化设置，如主题、语言等。
- 缓存数据： 存储一些需要频繁访问的数据，如 API 响应结果等。
- 表单数据

# sessionStorage

## 特点

- 存储容量： 每个域名下大约可以存储 5MB（具体大小因浏览器而异）。
- 生命周期： 数据在会话结束时（通常是关闭浏览器）自动过期。
- 作用域： 只能在同一个域名和协议下访问（不同子域名间不可共享）。

## 用途

- 一次性数据： 存储一些需要在会话期间保持的数据，如临时表单数据。
- 当前会话状态
- 跨页面传递数据

# indexDB

## 特点

- 存储容量： 可以存储大量数据，通常是 5MB 到 250MB。
- 生命周期： 数据可以持久存在，即使在浏览器关闭后也可以访问。
- 作用域： 可以在不同域名和协议下访问。

## 用途

- 离线应用： 例如，PWA（渐进式网页应用）可以使用 IndexDB 存储用户的离线数据，确保在没有网络连接时仍能使用应用。
- 复杂结构的数据： 例如，存储大型 JSON 数据或二进制文件。
- 离线数据存储
